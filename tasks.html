Homework 1: Tasks for to124

Your personal task is to develop a system for malls and their shops.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Shop. With the following private fields:

  1. String boss
  2. String headquarters
  3. String industry
  4. int founded
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Mall. With the following private fields:

  1. int id
  2. String name
  3. String website
  4. List<Shop> shops
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Mall> malls in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one mall with two shops. [4 marks]

Create controller co2123.hw1.controller.MallController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/malls" should show view malls/list. [2 marks]
    • Add the list of all malls from co2123.hw1.Hw1Application to the model as attribute malls. [2 marks]
    • Create the corresponding JSP to list the name and website of each mall followed by a link to URL "/shops?mall={id of mall}", e.g., "/shops?mall=3". [4 marks]
  2. Requests for URL "/newMall" should show view malls/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="mall" and action="/addMall" and inputs for all fields of class Mall except for shops. [5 marks]
  3. POST requests for URL "/addMall" should add a new mall to the list of all malls in co2123.hw1.Hw1Application and redirect to /malls. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.ShopController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/shops" should show view shops/list. [2 marks]
    • Based on request parameter mall determine the corresponding mall from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all shops of that mall to the model as attribute shops. [2 marks]
    • Create the corresponding JSP for the view to list the boss, headquarters, industry, and founded of each shop. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newShop?mall={id of mall whose shops are listed}", e.g., "/newShop?mall=3". [4 marks]
  2. Requests for URL "/newShop" should show view shops/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="shop" and action="/addShop" and inputs for all fields of class Shop. [6 marks]
    • Also add the request parameter mall with the ID of the current mall. [4 marks]
  3. POST requests for URL "/addShop" should add a new shop to the list of shops of the mall identified by request parameter mall and then redirect to /malls. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Mall and make it available in controller MallController. [4 marks]

When adding a new mall validate that

  1. The value of field id is unique within the list Hw1Application.malls. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field website is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Shop and make it available in controller ShopController. [4 marks]

When adding a new shop validate that

  1. The value of field boss is not empty. [2 marks]
  2. The value of field headquarters is not empty. [2 marks]
  3. The value of field industry is either "Clothes", "Footwear", or "Jewellery". [3 marks]
  4. The value of field founded is between (including) 1950 and 2024. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
